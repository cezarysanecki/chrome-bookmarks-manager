Bookmarks Manager

Privacy Policy

Last updated: April 2026  ·  Applies to Chrome extension version 1.0.0 and later

Summary: Bookmarks Manager stores all data locally on your device using chrome.storage.local. No personal data is collected, transmitted, or shared with any third party — except when you explicitly enable the optional AI feature and provide your own OpenAI API key.

1. What data we store

All data is stored exclusively in your browser's local storage (chrome.storage.local) and never leaves your device unless noted below.

KeyWhat it containsPurpose
bm_settings Feature flags (favicons, AI toggle), OpenAI API key (if provided) Persist user preferences
bm_stats Open count and last-opened timestamp per bookmark ID Sort by popularity, detect stale bookmarks
bm_history Last 30 bookmark operations (add / edit / delete) with timestamps Undo / audit trail
bm_view_prefs UI preferences: grid mode, group mode, sort order Restore view between sessions
bm_installed_at Timestamp of first install Baseline for stale-bookmark detection
bm_pending_add Title and URL of a bookmark being added via keyboard shortcut Pass data from background service worker to popup (cleared immediately after use)

2. Permissions used

PermissionWhy it's needed
bookmarks Read, create, edit, and delete Chrome bookmarks via the native API
storage Save user preferences and statistics locally via chrome.storage.local
tabs Read the URL and title of the active tab when adding a bookmark via keyboard shortcut (Ctrl+Shift+D)
host_permissions: <all_urls> Send HTTP HEAD requests to bookmarked URLs to check whether they are still reachable (dead-link detection). No page content is read — only the HTTP status code is inspected. Requests are initiated by the user opening the full-page view.

3. Optional AI feature

The extension includes an optional AI-assisted tag suggestion feature powered by OpenAI. This feature is disabled by default and requires you to:

  • Explicitly enable it in Settings, and
  • Provide your own OpenAI API key.

When enabled, the title and URL of a bookmark are sent to the OpenAI API (api.openai.com) solely to generate tag suggestions. No other data is included. Your API key is stored locally in bm_settings and is never transmitted to anyone other than OpenAI. OpenAI's own privacy policy applies to that interaction.

4. Data sharing

We do not collect, sell, rent, or share any of your data with third parties. No analytics, crash reporting, or telemetry services are used. No data is transmitted to the extension developer.

5. Data retention & deletion

All locally stored data is retained until you remove it. You can clear all extension data at any time by going to chrome://extensions, finding Bookmarks Manager, and clicking Remove, or by clearing site data for the extension in Chrome settings.

6. Children's privacy

This extension does not knowingly collect any information from children under the age of 13. It does not involve any data collection beyond what is described in this policy.

7. Changes to this policy

If this policy changes materially, the updated version will be published at this URL and the extension version number will be incremented. Continued use of the extension after such changes constitutes acceptance of the updated policy.

8. Contact

If you have questions about this privacy policy, please open an issue in the extension's source repository or contact the developer directly via the Chrome Web Store developer page.
